# Правила работы над репозиторием

1. Все взаимодействие с пользователем, внутренние обсуждения и документация (включая README.md) ведутся строго на русском языке.
2. Перед каждым `git push` обязателен ручной аудит на наличие секретов: паролей, ключей API, IP-адресов и любой конфиденциальной информации в актуальном коммите и истории ветки.
3. Конфигурационные данные, требующие секретности, хранятся только в локальных `.env` файлах или секрет-менеджере. Шаблоны с плейсхолдерами публикуются в репозитории вместо реальных значений.
4. При обнаружении утечки секретов немедленно перевыпускаем значения, удаляем компрометированные данные из истории Git и информируем ответственную команду.
