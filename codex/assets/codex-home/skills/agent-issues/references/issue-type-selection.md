# Issue type selection (ITIL/ITSM guided)

Сначала проверь допустимые `issue.type` в AGENTS.md проекта. Если список ограничен — выбирай ближайший по смыслу и уточняй при сомнениях.

## Быстрая матрица (ITIL/ITSM → типы issues)
- **Incident** (сервис деградировал/сломался, нужно восстановить работу) → `bug`
- **Service Request** (стандартный запрос: доступ/настройка/разовый сервис) → `task`
- **Change** (плановое изменение конфигов/инфры/процессов) → `chore`  
  - если изменение добавляет новую функциональность → `feature`
- **Problem** (устранение первопричины инцидентов) → `chore` или `task` (исследование/фикс)
- **Feature** (новая ценность/поведение продукта) → `feature`
- **Epic** (крупная инициатива из нескольких задач) → `epic`
- **Merge Request / PR only** (чистая интеграция/слияние) → `merge-request`
- **Molecule** → использовать только если проект определяет этот тип явно

## Правила выбора (коротко)
1. **Есть авария/деградация?** → `bug`.
2. **Новая функциональность?** → `feature` (или `epic`, если большой объём).
3. **Техдолг/рефактор/инфра/обслуживание?** → `chore`.
4. **Небольшое выполнение без изменения продукта?** → `task`.

## Примечания
- Не ставь `task` по умолчанию: тип должен отражать характер работ.
- Если тип не очевиден — задай уточняющий вопрос владельцу или укажи `UNCONFIRMED` в леджере.
