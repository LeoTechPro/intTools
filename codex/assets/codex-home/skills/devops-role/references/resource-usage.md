# Resource usage (local host safeguards)
- Перед тяжёлыми операциями проверь нагрузку: `top`/`htop`/`ps`/`docker stats` или аналог. Если высокая — согласуй перед запуском.
- Фиксируй использование ресурсов только при реальной дополнительной нагрузке (CPU/RAM/IO) и только если требуется проектом. Для machine-monitoring текущий канонический срез хранится в `/2brain/Resources/systems/Machine.vds.intdata.pro.md`, а runtime-state Probe Monitor — в `/int/probe/state/`.
- Не меняй системные лимиты/конфиги без явного запроса владельца.
- Не запускай нагрузочные тесты/билды параллельно без оценки ресурса.
- Для Docker/compose учитывай наличие внешней БД; не поднимай лишние контейнеры, если порт 5432 занят — ищи внешний кластер.
