# GetCourse

## Когда использовать

Используй этот навык, когда нужно:
- работать с API GetCourse в проекте `punctb.pro`;
- проверить/описать возможности API-ключа без изменения данных;
- быстро найти актуальные официальные материалы по GetCourse API.

## Границы безопасности

- По умолчанию работай в режиме **read-only**.
- Не выполняй операции создания/редактирования (`add`, `update`, изменение статусов), если пользователь явно не попросил.
- Не печатай реальные ключи в выводе, логах и коде.

## Где лежат данные проекта punctb.pro

- Локальные ключи интеграций хранятся в: `/git/punctb/backend/.env`.
- Переменная Bitrix24: `BITRIX_WEBHOOK_BASE_URL`.
- Переменная GetCourse LMS: `GETCOURSE_LMS_PUNCTB_PRO_API_KEY`.
- Значения секретов не коммитить и не дублировать в markdown/репозиторий.

## Обезличенный безопасный workflow

1. Убедись, что ключ присутствует в `.env` (выводи только имя переменной, без значения).
2. Сверь тип ключа в интерфейсе GetCourse: `https://<ACCOUNT_DOMAIN>/saas/account/api`.
3. Для безрисковой диагностики используй только экспортные/read-only методы.
4. Если требуется тест API-запроса, начинай с чтения групп:

```bash
curl -sS "https://<ACCOUNT_DOMAIN>/pl/api/account/groups?key=<API_KEY>"
```

5. Для детального экспорта используй двухшаговый механизм `users/deals/payments -> exports/{export_id}` из официальной документации.
6. Любые write-операции выполняй только после явного подтверждения пользователя.

## Что обычно можно делать с API-ключом

- `read-only` ключ: чтение/экспорт данных API без создания/редактирования объектов.
- `read+write` ключ: полный доступ к API-операциям (включая создание/изменение пользователей, заказов и статусов).
- Отзыв доступа выполняется удалением ключа в `.../saas/account/api`.
