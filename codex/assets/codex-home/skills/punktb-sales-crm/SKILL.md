# Пункт Б: продажи и CRM

Использовать постоянные подключения `amocrm`, `getcourse` и `bitrix24-rest`. Для коммуникаций использовать Umnico-tools внутри `amocrm`.

## Рабочий порядок

1. Определить клиента по нормализованному телефону, затем по email; имя использовать только как дополнительный признак.
2. Прочитать контакт, связанные сделки, воронку, задачи, заметки и историю до изменения данных.
3. Отделить факты CRM от предположений и предложить следующее действие.
4. При запросе пользователя выполнить разрешённое изменение и перечитать изменённый объект.
5. Для сообщения сначала найти Umnico-диалог, прочитать канал и историю, подготовить окончательный текст и проверить адресата.
6. Отправлять только после явного подтверждения конкретного текста и канала; технического параметра `confirm_send` у коннектора нет, поэтому не вызывать send-tool до подтверждения.

## Стоп-условия

- Не угадывать ID воронки, этапа, поля, менеджера, интеграции, канала или диалога.
- Не отправлять тестовые сообщения реальным клиентам.
- Не выполнять массовые изменения или рассылки без отдельного точного подтверждения охвата.
- Не выводить API-ключи, пароли, cookies, токены и содержимое secret stores.
- Не считать amoCRM API методом отправки email: использовать подключённый коммуникационный канал или сообщить об отсутствии mailbox-интеграции.
- Если записи звонков доступны только ссылкой с истёкшей авторизацией, использовать браузерный amoCRM-сеанс либо запросить отдельную выборку; не объявлять аудио изученным без воспроизведения.
