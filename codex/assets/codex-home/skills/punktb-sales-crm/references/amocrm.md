# amoCRM

## Чтение и анализ

- `contacts_search` → `contacts_get(with_related=...)` для идентификации клиента.
- `associations_get_linked` для связи контакта со сделками и компаниями.
- `leads_get`, `notes_list`, `tasks_list`, `analytics_get_events` для полной хронологии.
- `pipelines_list` и `pipelines_list_statuses` перед интерпретацией этапа.
- `account_list_custom_fields` перед записью нестандартного поля.

## Изменения

- Контакт: `contacts_update`.
- Сделка и этап: `leads_update`.
- Примечание: `notes_create`.
- Задача и завершение: `tasks_create` / `tasks_update`.
- Входящий неразобранный лид: сначала прочитать, затем `unsorted_accept` или `unsorted_reject` только по явному решению.

После каждого изменения перечитать объект и назвать его ID, изменённые поля и фактическое состояние. Не создавать дубли: перед созданием искать по телефону и email.

## Квалификация

Использовать поля и этапы, возвращённые живым аккаунтом. Не переносить значения из старой Bitrix24 автоматически. При конфликте истории amoCRM и Bitrix24 фиксировать оба источника и дату события.
